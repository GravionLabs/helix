Dynamic Form — Advanced Features
  • Custom widget — the score renders a PrimeNG rating registered as widget: 'rating' via provideHelixDynamicForms({ widgets }).
  • Conditional states — non-draft status makes the title readonly, "archived" disables the score, and escalating makes the reason required.
  • Central error resolver — required/e-mail messages come from a route-provided errorMessageResolver.
  • Schema-derived defaults — no [model] input; the initial value comes from the schema's .default() calls.
  • extraSchema escape hatch — the reviewer e-mail carries a raw signal-forms validate rule (must end in @example.com).

Decorated domain schema

This form uses a plain Zod domain schema (as it would ship in a shared package) that is decorated with UI metadata after the fact: helixMeta(TaskSchema.shape.field, { ... }) attaches to the exact field instances, and the conditional callbacks are typed with z.infer<typeof TaskSchema> — no hand-written model interface.
